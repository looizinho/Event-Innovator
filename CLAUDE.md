## Project Overview

Event Innovator is a pnpm workspace monorepo with a React frontend (portfolio) and Express backend (api-server), using PostgreSQL for data persistence. The project is configured to run on Replit with strict supply-chain security controls.

## Architecture

**Three artifact packages** (deployable/runnable services):
- `artifacts/api-server` — Node.js Express API server (port 5000 in dev)
- `artifacts/portfolio` — React SPA frontend with Vite (configured via env vars)
- `artifacts/mockup-sandbox` — Vite React sandbox (unused template)

**Four library packages** (shared code):
- `lib/db` — Drizzle ORM schema, migrations, and database utilities. Exports `./src/index.ts` and `./src/schema/index.ts`
- `lib/api-spec` — OpenAPI spec (`openapi.yaml`) and Orval codegen config that generates both `api-zod` and `api-client-react`
- `lib/api-zod` — Generated Zod schemas from OpenAPI spec (regenerated by `pnpm run codegen`)
- `lib/api-client-react` — Generated React Query hooks from OpenAPI spec (regenerated by `pnpm run codegen`)
- `lib/integrations/*` — Scoped library packages (configured in workspace but empty)

**scripts** — TypeScript utility scripts (e.g., `pnpm run hello`)

## Key Technologies

- **Runtime**: Node.js 24, TypeScript 5.9, pnpm workspaces (catalog pattern for dependency versioning)
- **Backend**: Express 5, Pino logging, Drizzle ORM v0.45.2
- **Database**: PostgreSQL via `pg` driver, migrations via `drizzle-kit`
- **Frontend**: React 19.1.0 (exact version required by Expo), Vite 7.3.2, Tailwind CSS 4, Wouter routing, React Query v5, Radix UI, TailwindCSS plugins (`@tailwindcss/vite`, `@tailwindcss/typography`)
- **API**: OpenAPI spec with Orval v8.9.1 for codegen (generates both Zod schemas and React hooks)
- **Validation**: Zod v3.25.76, `drizzle-zod` for auto-generating Zod from Drizzle schemas

## Common Commands

```bash
# Typecheck and build everything
pnpm run build

# Typecheck only (faster iteration)
pnpm run typecheck

# Run API server in development (rebuilds then starts on port 5000)
pnpm --filter @workspace/api-server run dev

# Run portfolio frontend (Vite dev server, port from BASE_PATH env var)
pnpm --filter @workspace/portfolio run dev

# Regenerate API types and React hooks from OpenAPI spec
# (updates lib/api-zod and lib/api-client-react)
pnpm --filter @workspace/api-spec run codegen

# Push database schema changes to PostgreSQL (dev only)
pnpm --filter @workspace/db run push
pnpm --filter @workspace/db run push-force  # Force push (destructive)
```

## Environment Variables

**Required for all development:**
- `DATABASE_URL` — PostgreSQL connection string (e.g., `postgres://user:pass@localhost:5432/dbname`)
- `PORT` — API server port (default 5000 in dev)
- `BASE_PATH` — Frontend base path for Vite (e.g., `/portfolio`)

## Important Files

- **`pnpm-workspace.yaml`** — Workspace definition, catalog versions, security settings (minimumReleaseAge: 1440 min)
- **`lib/db/drizzle.config.ts`** — Database config pointing to `DATABASE_URL`
- **`lib/api-spec/openapi.yaml`** — Source of truth for API contract
- **`lib/api-spec/orval.config.ts`** — Codegen config (generates to `lib/api-zod` and `lib/api-client-react`)
- **`artifacts/api-server/build.mjs`** — esbuild configuration (bundles to ESM, externals configured)
- **`artifacts/portfolio/vite.config.ts`** — Frontend build config with Replit plugins

## Database Schema

Database schema is defined in `lib/db/src/schema/index.ts` and each table should:
1. Define a Drizzle `pgTable` (e.g., `postsTable`)
2. Export an insert schema with `createInsertSchema` from `drizzle-zod`
3. Export inferred types (`InsertPost`, `Post`)

Example pattern (see comments in schema file):
```typescript
export const postsTable = pgTable("posts", { id: serial("id").primaryKey(), ... });
export const insertPostSchema = createInsertSchema(postsTable).omit({ id: true });
export type InsertPost = z.infer<typeof insertPostSchema>;
export type Post = typeof postsTable.$inferSelect;
```

Push schema changes with `pnpm --filter @workspace/db run push`.

## API Development Workflow

1. **Update OpenAPI spec** — Edit `lib/api-spec/openapi.yaml` to define new endpoints, request/response schemas
2. **Regenerate types** — Run `pnpm --filter @workspace/api-spec run codegen`
   - Updates `lib/api-zod/src/generated/*` (Zod schemas)
   - Updates `lib/api-client-react/src/generated/*` (React Query hooks)
3. **Implement endpoint** — In `artifacts/api-server/src`, create route handler using generated Zod schema for validation
4. **Use in frontend** — Import generated hook from `@workspace/api-client-react` in React components

The API runs on port 5000 (dev) and serves routes under `/api`.

## TypeScript Configuration

- **Base config** (`tsconfig.base.json`):
  - Target: `es2022`, module: `esnext`, strict mode enabled
  - `noEmitOnError`, `noImplicitAny`, `strictNullChecks`, `strictPropertyInitialization`
  - `moduleResolution: "bundler"` for bundler-era resolution
  - Custom condition: `"workspace"` for workspace exports
- Each package has its own `tsconfig.json` that extends base

## Frontend Patterns

- **Routing**: Uses `wouter` (lightweight client-side router) with base path support
- **State**: React Query for server state, React Context for UI state
- **Components**: Radix UI + Tailwind CSS for UI primitives and styling
- **Forms**: React Hook Form + Zod validation (via generated schemas)
- **Environment**: `import.meta.env.BASE_URL` for base path, `import.meta.env.VITE_*` for custom env vars

## Backend Patterns

- **Logging**: Pino logger with pino-http middleware for request logging
- **CORS**: Enabled for all origins (configurable in `app.ts`)
- **Request body**: Express JSON/URL-encoded middleware
- **API structure**: Routes mounted at `/api` with individual route files in `src/routes/`

## Gotchas & Important Details

1. **React/React-DOM version lock**: Both are pinned to `19.1.0` (exact) because Expo requires it—never auto-update
2. **minimumReleaseAge security**: pnpm enforces 1440-minute delay on all npm packages (see `pnpm-workspace.yaml`). Helps defend against supply-chain attacks. Only exclude trusted packages (e.g., @replit/*, typescript)
3. **esbuild for backend**: `build.mjs` bundles to ESM with specific externals—check if adding new deps (especially native modules)
4. **Orval codegen**: Always run after OpenAPI spec changes; codegen is the source of truth for frontend types
5. **DATABASE_URL required**: Both dev server and migrations require this env var—will error if missing
6. **Port collisions**: Replit port mapping is configured; if running locally, adjust ports in vite.config.ts and build.mjs
